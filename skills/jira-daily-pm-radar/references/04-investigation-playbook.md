# Investigation Playbook

When a PM asks “why was this issue flagged?”:

1. Use `jira-radar issue <ISSUE_KEY>` or MCP `issue_context`.
2. Check status, updated date, assignee, priority, labels, epic/parent.
3. Check latest comments.
4. Check changelog for status/sprint/priority changes.
5. Answer with evidence.

When a PM asks “what should I do now?”:

1. Read `action-list.json`.
2. Group by daily / refinement / cleanup.
3. Keep it short.
4. Do not suggest mass changes unless explicitly requested.
